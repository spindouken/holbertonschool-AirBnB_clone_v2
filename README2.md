# HBNB 2.0: AirBnB Clone - Version 2
This console serves as the second component of the AirBnB project, a comprehensive program developed by Holberton School to teach students essential components of higher-level programming. The AirBnB project aims to provide a hands-on learning experience for students by simulating the development of a simplified version of the popular online accommodation platform, AirBnB, which we lovingly call HBnB.

The console component of this project plays a crucial role in facilitating the development of the HBnB server. It provides a command-line interface that enables developers to interact with the server and perform various operations such as creating and managing objects, performing CRUD (create, read, update, and delete) operations on data, and querying data from the server's database.

This project helped us gain a deeper understanding of higher-level programming concepts such as object-oriented programming, database management, and web development.

### Functionalities:
* Create a new object (ex: a new User or a new Place)
* Retrieve an object from a file, a database etc...
* Do operations on objects (count, compute stats, etc...)
* Update attributes of an object
* Destroy an object

## Environment
This project is interpreted/tested on Ubuntu 20.04 LTS using python3

## Installation
* Clone this repository: `git clone "https://github.com/spindouken/holbertonschool-AirBnB_clone_v2.git"`
* Access AirBnb directory: `cd AirBnB_clone`
* Run hbnb(interactively): `./console` and enter command
* Run hbnb(non-interactively): `echo "<command>" | ./console.py`

## File Descriptions
:video_game: [console.py](console.py) :video_game:
#### (contains the entry point of the command interpreter) 
List of Supported Commands:
* `EOF` - Exits the console. 
* `quit` - Exits the console.
* `<emptyline>` - Overwrites default emptyline method and does nothing.
* `create` - Creates an object of any class with the given attributes.
* `destroy` - Deletes a specified object. 
* `show` - Prints the string representation of an instance based on the class name and id.
* `all` - Prints all string representation of all instances based or not on the class name.
* `update` - Updates an instance based on the class name and id by adding or updating attribute (save the change into the JSON file). 

### :computer: `models/` :computer:
#### (contains classes used in the project)
[base_model.py](/models/base_model.py) - The BaseModel class from which future classes will be derived.
* `def __init__(self, *args, **kwargs)` - Initialization of the base model.
* `def __str__(self)` - String representation of the BaseModel class.
* `def save(self)` - Updates the attribute `updated_at` with the current datetime.
* `def to_dict(self)` - returns a dictionary containing all keys/values of the instance.

Classes inherited from Base Model:
* [amenity.py](/models/amenity.py)
* [city.py](/models/city.py)
* [place.py](/models/place.py)
* [review.py](/models/review.py)
* [state.py](/models/state.py)
* [user.py](/models/user.py)

### :floppy_disk: `/models/engine` :floppy_disk:
#### (contains File Storage & DB Storage classes)
[file_storage.py](/models/engine/file_storage.py) - serializes instances to a JSON file & deserializes back to instances
* `def all(self)` - returns the dictionary __objects
* `def new(self, obj)` - sets in __objects the obj with key <obj class name>.id
* `def save(self)` - serializes __objects to the JSON file (path: __file_path)
* ` def reload(self)` -  deserializes the JSON file to __objects
